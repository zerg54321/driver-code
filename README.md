# 北京市交通违法查询与精准法条对齐系统 (Beijing Traffic Violation Query & Precise Law Alignment System)

这是一个基于 React + TypeScript + Tailwind CSS 构建的高性能、高颜值的交通违法行为精准查询与法条深度对齐系统。系统旨在帮助驾驶员、执法人员及法律研究者快速检索各类交通违法行为，并**精准、去噪**地对齐到最权威的法律条款（国家法律、实施条例及北京市地方性法规）。

---

## 🚀 项目概述

在日常交通管理与普法宣传中，传统的交通违法代码与法条查阅往往存在“条文冗长”、“检索困难”以及“地方标准与国家法律混淆”等痛点。

本项目通过将结构化的交通违法行为数据与完整的**三大法条数据库**进行多维度交叉关联：
1. **国家级法律**：《中华人民共和国道路交通安全法》（简称“法”）
2. **行政法规**：《中华人民共和国道路交通安全法实施条例》（简称“条”）
3. **地方性法规**：《北京市实施〈中华人民共和国道路交通安全法〉办法》（简称“北”）

通过创新的**精准法条对齐算法**，系统能自动定位并截取至具体的**“条、款、项”**，剔除无关条文，实现法律信息的高效降噪与直观呈现。

## 🛠️ 核心业务逻辑

项目的底层技术架构和业务处理流程主要由以下模块构成：

### 1. 数据解析与状态管理 (`src/data/`)
* **CSV 格式化底层**：在 `rawCsv.ts` 中维护了完整的北京市交通违法数据集，由于部分数据包含逗号及复杂的双引号字符，系统在 `lawsData.ts` 中实现了一个健壮的 CSV 行解析器 `parseCSVLine`，支持转义与嵌套，确保字段读取百分之百准确。
* **数据懒加载缓存**：在首屏加载时，`getViolationsList` 自动对原始 CSV 进行读取，解析为强类型的 `ViolationItem[]` 结构，并将其缓存至内存中，避免重复解析。
* **自动分类器**：根据违法行为描述的语义特征，系统自动将数据划分为五个大类：
  * 🚶 **行人和乘车人**
  * 🚲 **非机动车**
  * 🚗 **机动车通行**
  * 🛣️ **高速及快速路**
  * 🚦 **道路交通（常规通用）**

### 2. 精准法条对齐与智能去噪算法
这是本系统的核心技术壁垒。每一个违法行为的 `behaviorLaw` (行为条款) 和 `punishLaw` (处罚条款) 都包含结构化的引用字符串（例如 `law:90;bj:95,1`）。
* **法条解析器 (`parseLawRefs`)**：解析引用字符串，将其结构化为 `{ db: 'law' | 'reg' | 'bj', artNum: number, paraNum?: number, itemNum?: number }`。
* **分级索引匹配**：在 `lawsData.ts` 的 `getPreciseLawText` 中，算法先定位到具体的条（`art`），再根据是否有特定的**款（Paragraph）**或**项（Item）**，仅将匹配的子节点文字提取出来。
* **中文数字转化**：为了提供极致符合中文法律阅读习惯的体验，系统实现了 `toChineseNumeral`，自动将数字（如 `104`）转换为符合法典规范的传统中文数字（如 `一百零四`），用于渲染诸如“第一百零四条第二款第（一）项”的精确小标题。

### 3. 用户交互与多端适配界面 (`src/components/`)
* **统一搜索控制台**：支持拼音、中文关键字、违法代码、记分分值（0/1/3/6/9/12分）以及罚款金额的复合模糊搜索。
* **双栏对齐对比视图 (`DetailView`)**：大屏下采用分栏布局，左侧清晰展示违法事实与处罚措施，右侧将**“认定违法事实法条”**与**“处罚救济法条”**双向平铺对齐，重点句段加粗显示，法律逻辑一目了然。
* **响应式移动端抽屉 (`MobileDetailModal`)**：针对移动端手势和窄屏进行了专门适配，采用沉浸式弹出面板，支持顺滑的滚动体验。

## 🔮 后期需要完善与优化展望

为了进一步提升系统的专业度、应用场景和用户体验，后续可从以下几个维度进行迭代和深度打磨：

### 1. 🔍 检索体验升级 (Fuzzy Search & Pinyin)
* **痛点**：目前采用简单的 `String.includes` 检索，无法处理拼音首字母缩写、错别字或近义词（例如搜索“超速”可能漏掉描述为“超过规定时速”的条目）。
* **优化方向**：
  * 引入 `Fuse.js` 进行轻量级客户端模糊搜索。
  * 建立交通法律领域专属的**同义词词典**（如“非机动车”与“自行车/滑板车”、“违停”与“违反停放规定”）。
  * 增加拼音索引，支持输入拼音首字母快速定位违法行为。

### 2. 🧠 AI 交通法规助手 (Gemini 2.5/3.5 SDK 深度融合)
* **应用场景**：用户可以像咨询真人律师一样，用自然语言提出问题（如：“我在东三环辅路没带安全带被拍了，要扣几分罚多少钱？”、“我收到一条代码1344的短信，怎么解读？”）。
* **技术实现**：
  * 在 Express 服务端（`server.ts`）集成 `@google/genai` 官方 SDK。
  * 配合 **RAG (检索增强生成)**：AI 根据用户输入，首先在本地违法库和三大法条 JSON 数据库中检索最相似的几条数据作为背景知识，作为 Context 喂给 Gemini 模型，确保生成绝对专业、真实、不产生幻觉的法律解答。

### 3. 📊 交通安全大屏与可视化统计 (Data Visualization)
* **应用场景**：为交管部门或社会机构提供北京市交通违法的分布及危害性研究。
* **优化方向**：
  * 引入 `Recharts` 或 `D3.js`。
  * 绘制**违法行为扣分分布比例图**、**各类别高频违法行为罚款区间雷达图**。
  * 违法代码规律分析（例如代码第一位代表违法分类，可以通过图表直观展示各大类的占比）。

### 4. 💾 个人驾照分扣减模拟器与收藏夹 (IndexedDB)
* **应用场景**：普通车主常备工具。
* **优化方向**：
  * **记分模拟器**：用户可将多个违法行为加入“模拟扣分板”，实时累计扣分和罚款金额，当达到 12 分时触发红色警告，并给出对应的科目一重考等救济法条提示。
  * **我的收藏**：使用浏览器 `localStorage` 或 `IndexedDB` 存储车主关注的常见行为（如“限行”、“违停”），方便随时快速查阅。
